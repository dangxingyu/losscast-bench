# Test Case: nanochat d12

A small nanochat model (12 layers, 768 dim) for testing the code-rewriter conversion pipeline.

## Source
- `source/gpt.py` — nanochat GPT model (from karpathy/nanochat master)
- `source/optim.py` — MuonAdamW hybrid optimizer
- `source/base_train.py` — training script with CLI args

## Simulated Training Config
Simulates a nanochat d12 run on 8x H100 GPUs:

```
depth=12, aspect_ratio=64 → d_model=768, n_heads=6
batch_size=32 per device × 8 GPUs × 2048 seq_len = 524288 tokens/step
matrix_lr=0.02 (Muon), embedding_lr=0.3, unembedding_lr=0.008 (AdamW)
warmup=40 steps, warmdown_ratio=0.65
eval_every=250 steps
```

## What Conversion Means

The code-rewriter **decouples** the model from the nanochat framework — it does NOT simplify or approximate the architecture. All custom features must be preserved exactly:

- ReLU² activation (not GELU, not ReLU)
- QK norm with 1.2× sharpening
- Value embeddings on alternating layers with input-dependent gates
- Smear gate (bigram mixing of previous token embedding)
- Backout lambda (subtract mid-layer residual)
- Per-layer resid_lambdas and x0_lambdas
- Sliding window attention pattern (SSSL)
- RoPE with theta=100000
- Logit softcap at 15

What gets removed/replaced:
- `nanochat.*` imports → inline the code
- `flash_attn` → `F.scaled_dot_product_attention` (same math, portable)
- Distributed code (DDP, rank checks) → single-device
- Inference paths (KV cache, generate) → training forward only
- FP8 paths → removed (precision is a training detail, not architecture)

## Expected Conversion Output
- `expected/config.json` — structured config
- `converted/config.json` — code-rewriter output (generated by agent)
- `converted/model.py` — standalone model with all features preserved
- `converted/optimizer.py` — standalone MuonAdamW
