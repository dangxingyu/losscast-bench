Two ways to join the loop.

LossCast-Bench is both a place to publish real pretraining curves and a benchmark for predictors that learn from them.

I run pretraining.

Contribute a recipe, loss curve, and optional code snapshot from a real run. The benchmark checks the schema and adds it to the public train/val pool.

Send your agent this instruction:
Read https://raw.githubusercontent.com/dangxingyu/losscast-bench/master/CONTRIBUTE.md and follow the instructions to submit a training run to LossCast-Bench.
  • Package config.json and losses.json for each run.
  • Validate locally before opening a data PR.
  • Grow the benchmark while keeping hidden test runs private.
Contribute a run

I build predictors.

Train on public runs, tune on public validation, and submit a predictor that maps a full training recipe to a full loss trajectory.

Send your agent this instruction:
Read https://raw.githubusercontent.com/dangxingyu/losscast-bench/master/PREDICT.md and follow the instructions to build and evaluate a loss predictor for LossCast-Bench.
  • Load train/val splits with the package API.
  • Forecast every required eval checkpoint.
  • Compare locally, then target the hidden d26 leaderboard.

One-minute local demo.

Install the package, generate Chinchilla predictions on public validation, and score the complete prediction grid.

git clone https://github.com/dangxingyu/losscast-bench.git
cd losscast-bench
pip install -e .

python -c "
from losscast_bench.data import load_split
from losscast_bench.baselines.chinchilla import predict_batch
from losscast_bench.schema import save_predictions

configs, _ = load_split('val')
preds = predict_batch(configs)
save_predictions(preds, 'chinchilla_predictions.json')
"

losscast-eval --split val -p chinchilla_predictions.json
